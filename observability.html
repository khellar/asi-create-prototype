Boundary · note 3   The inspector is read-mostly. The only write actions are 👍 keep / 👎 forget on a memory (human-curated purge, which the glacier preserves) and "pin to view." The agent does its own diffing and forming; the user watches it happen and occasionally curates. No "edit memory" form — that would make the user the agent's bookkeeper.

§4 · Note 7 — click the ⋯ to see the actual prompt, reasoning, tool-calls, episodes

The cycle inspector

This is note 7 in full. Clicking the ⋯ on any cycle (in the feed or the metric cards) opens the cycle's complete trace: the exact prompt that was sent (the scaffolding-check angle), the model's private reasoning, the tool-calls, and the episodes referenced from history.metta. click any feed row in §1 — it really opens →

Scaffolding-check · note 7   The whole point of surfacing the raw prompt is to audit how much the harness is steering the model. The inspector tags each prompt block by source — system / self-model / world-state / recalled-memory / board / instructions — and shows a scaffold ratio: how many prompt chars are framework-injected vs. genuinely the agent's own context. Khellar's morning thesis ("the ~74k-char prompt is heavy-handed") becomes a measurable number per cycle.
§5 · Note 5 — Cmd-K + slash commands

Report a bug · report suspected emergent behavior

A command palette (Linear/Vercel pattern) bound to ⌘K. The two flagship commands — /report-bug and /report-emergent — bundle the full context windows of the last 20 cycles (real promptcap-*.txt files + reasoning + tool-calls), so the team can read the actual prompt, check for scaffolding, and evaluate. press ⌘K now, or click → Open palette

Why bundle 20 cycles · note 5   A single screenshot can't show whether "emergent behavior" was genuine or prompted. 20 full context windows let a reviewer trace the lineage: what was injected, what the model added, when a belief or strategy first appeared. The bundle is a .zip of the raw promptcap files + the matching mind_feed entries + the diffs — submitted to the rig's "Notes & advice" thread or a private report queue.
